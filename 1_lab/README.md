# itmo-containers

## Когда НЕ стоит использовать контейнеры

1) **Приложения с GUI**, требующие прямого доступа к графической подсистеме и аппаратному обеспечению (например, CAD-системы, видеоредакторы)
2) **High-Performance Computing (HPC)** приложения, где даже минимальные накладные расходы на виртуализацию критичны для производительности (например, научные вычисления с CUDA, требующие прямого доступа к GPU)

## Плохой доцкер файл

1) Используется тег `latest`
2) Всё делается из-под root пользователя 
3) Очень много слоев, образ весит много
4) Отсутствует EXPOSE
5) Отсутствует VOLUME для персистентных данных

## Хороший доцкер файл

1) Устанавливается определенная версия (5)
2) Установка пакетов под root, дальше идет переключение под user-1000
3) Слои скомпанованы
4) Установка библиотек вынесенный в отдельный файл
5) EXPOSE присутствует
6) VOLUME определен для персистентных данных
